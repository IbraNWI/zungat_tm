# zungat_tm

Интеграция между Bitrix24 и TaxiMaster. Проект работает как локальное приложение Bitrix24 через Django и поддерживает OAuth-авторизацию со стороны Bitrix24.

## Запуск на сервере

1. Скопируйте пример окружения:

   ```bash
   cp .env.example .env
   ```

2. Заполните `.env`: обязательно поменяйте `DJANGO_SECRET_KEY`, домены в `DJANGO_ALLOWED_HOSTS` и `DJANGO_CSRF_TRUSTED_ORIGINS`, а также временный пароль администратора.

3. Запустите контейнер:

   ```bash
   docker compose up -d --build
   ```

При старте контейнер автоматически выполнит проверку Django, применит миграции, соберет статику и при указанных `DJANGO_SUPERUSER_*` создаст первого администратора.

Статика приложения собирается в `/app/staticfiles` и раздается Django через WhiteNoise. На хосте эта папка хранится как `./staticfiles`, поэтому файлы не пропадают при пересоздании контейнера.

Media-файлы хранятся в `/app/media` и монтируются в `./media`. Для боевого сервера их лучше отдавать через nginx или другой внешний прокси.

Проверка доступности:

```bash
curl http://127.0.0.1:8000/health/
```

После первого успешного запуска лучше удалить `DJANGO_SUPERUSER_PASSWORD` из `.env` и перезапустить контейнер.
